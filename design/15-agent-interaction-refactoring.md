# Agent 交互界面重构方案：蓝点与抽屉 (Bottom Sheet) 模型

## 1. 设计初衷与核心痛点
当前的 Agent 交互（浮窗 + 输入框蓝点 + 悬浮球）存在概念冲突、入口冗余、移动端体验差等问题。特别是在“文件视图”与“会话视图”频繁切换时，上下文容易丢失，浮窗遮挡严重。

**重构目标：**
- 废弃传统的悬浮窗（Floating Window）和右下角悬浮球。
- 引入**底部抽屉（Bottom Sheet）**作为伴随视图交互的主力容器。
- 以**输入框蓝点**作为抽屉和会话状态的唯一枢纽，精简入口。
- 实现“全屏主视图”与“伴随抽屉”之间的无缝互斥与流转，杜绝“双重套娃”。

---

## 2. 核心架构：蓝点状态机

蓝点不再只是一个简单的开关，它是当前工作区（Workspace）AI 上下文的指示器。

### 蓝点的三种状态

| 状态 | 触发场景 | 蓝点表现 | 行为含义 |
| :--- | :--- | :--- | :--- |
| **初始 (Initial)** | 刚启动应用，或点击了 `[+] 新建`。当前没有绑定任何已存在的 Session。 | **空心圆 / 灰色点** | “虚位以待”。此时发送消息，将基于当前上下文（如正在看的文件）立刻创建**新 Session**。点击蓝点弹出抽屉，内部展示历史列表供挑选。 |
| **待绑定 (Pending)** | 在初始状态下，从右侧列表（或抽屉内的列表）点击选中了某个旧 Session，但还**未发送任何消息**。 | **实心蓝点** | “已选中，可反悔”。抽屉/主视图展示该 Session 内容。此时若再次去列表点击其他 Session，绑定目标直接静默切换。一旦发消息，转入已绑定状态。 |
| **已绑定 (Bound)** | 只要在当前上下文中发送过消息（不论新建还是复用），该 Session 就被“死死绑定”。 | **高亮实心蓝点**<br>*(Agent 活跃时可变为呼吸灯/动画)* | “专属频道”。这是最核心的保护态：此时哪怕用户去列表点击其他 Session 查看（历史只读），底部的输入框和蓝点**依然绑定当前 Session**，不会被切走。保护正在进行的对话不被打断。 |

### 全局重置机制
在 ActionBar 左侧（或抽屉内首位）提供一个明确的 **`[➕ 新建]`** 按钮。无论在什么状态下，点击它即刻斩断当前绑定，回到 **状态1（初始）**，并清空抽屉/主视图。

## 3. 容器自适应互斥逻辑 (Adaptive Container Mounting)

这是解决“全屏视图”和“抽屉视图”打架问题的核心机制：**同一个绑定的 Session，同一时刻只能在一个容器中展示，自动“搬家”。**

### 场景 A：专注聊天（主视图 = 当前绑定 Session）
- **前提**：用户在全屏主视图中查看当前绑定的 Session（例如，刚从列表点开，或点击了“全屏”按钮）。
- **交互表现**：
  - **抽屉被禁用**：蓝点处于**不可点击**（或置灰/隐藏功能）状态。
  - **发消息**：在底部输入框输入并发送，**抽屉绝对不会弹出**。消息直接顺滑追加到主视图的全屏对话流中。
  - **体验**：标准的沉浸式聊天应用体验。

### 场景 B：伴随工作（主视图 = 文件 / 其他视图）
- **前提**：用户正在主视图阅读代码（如 `FileViewer`）或查看其他不相干的旧 Session 历史，但底部仍绑定着当前的活跃 Session。
- **交互表现**：
  - **抽屉被激活**：因为主视图被占用了，当前绑定的 Session **自动收纳到底部抽屉（蓝点）中**。蓝点可点击。
  - **发消息 / 点蓝点**：底部 **半屏抽屉 (Bottom Sheet)** 弹出。用户可以在不离开当前文件视图的情况下，在半屏抽屉里看 Agent 回复和继续交互。
  - **体验**：完美的“边看边聊”分屏伴随体验。

---

## 4. 抽屉 (Bottom Sheet) 的内部设计与流转

### 4.1 抽屉的形态
- **收起 (Closed)**：不显示。底部仅保留 ActionBar（输入框 + 蓝点）。
- **半屏 (Half)**：覆盖屏幕下半部（约 40%-50%），软键盘弹起时随之抬升。此时主视图上半部分（如文件内容）依然可见可读。

### 4.2 抽屉的收展操作
- **展开**：在场景 B 下，点击蓝点、点击输入框准备打字、或通过快捷键发送消息时自动弹出到半屏。
- **收起**：点击抽屉外部遮罩区域（主视图）、再次点击蓝点、或执行下滑手势。

### 4.3 抽屉内部结构
- **Header 区域**：
  - 标题（当前 Session 名称）。
  - **`[⛶ 全屏]` 按钮**：点击后，抽屉立刻收起，当前 Session 切换到**全屏主视图**展示（即转入场景 A）。
  - 下拉菜单（可选）：提供快速切换其他 Session 的快捷入口。
- **Content 区域**：
  - 核心展示区：复用 `<SessionViewer />` 组件，展示对话流式输出和记录。

---

## 5. UI 变更清单总结

1. **移除**：全局右下角悬浮球（FAB）。
2. **移除**：原有的 Floating Panel（浮窗）组件。
3. **新增**：`<BottomSheet />` 组件容器，用于包裹在半屏状态下需要渲染的 `<SessionViewer />`。
4. **修改 ActionBar**：
   - 强化蓝点（或将其做成一个明显的图标按钮）作为抽屉唤起唯一入口。
   - 实现蓝点的三态视觉表现（空心/实心/高亮呼吸）。
5. **重构路由/状态映射**：
   - 引入 `activeBoundSessionId` 全局状态。
   - 渲染判断：`if (mainViewType === 'session' && mainViewSessionId === activeBoundSessionId)` -> 在主视图渲染并禁用抽屉。
   - 渲染判断：`else` -> 激活抽屉，并在抽屉被触发时在其中渲染。
